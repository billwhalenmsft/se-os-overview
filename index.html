The economics

Tokenomics — scale the output, not the spend

Running one SE at full throttle this year took 2+ billion tokens a month. That number is not the cost of operating MCAPS OS — it's the cost of discovering it: one SE figuring out the patterns, skills, and plays from scratch. The whole design question is how every SE inherits that output without re-paying the discovery bill.

Discovery1 SE · ~2B tok/mo

Marginal costdrops toward zero per SE

The expensive part is paid once. The durable asset is the skill, not the conversation that produced it — so the next SE starts from the artifact, not the blank page.

How we hold the line

Six levers that decouple SE scale from token burn

01

Skills, not re-derivation

The output of every engagement is harvested into reusable skills, templates, and patterns. Agents load the artifact instead of re-discovering context every run — the single biggest spend killer.

02

Right model for the job

A cost ladder, not one big model for everything. Small models (e.g. Phi-4-mini) handle transcript parsing at ~15–20× lower cost; escalate to frontier models only for the hardest reasoning.

03

Curated memory + a promotion gate

An operational store remembers facts; only durable meaning is promoted to account narrative. Agents stop re-ingesting the same context — memory replaces repetition.

04

Deterministic where possible

Contracts, indexers, templates, and rules-based fallbacks do real work with zero tokens. The model is reserved for judgment, not plumbing the pipeline can do for free.

05

Estimate before you spend

A callable Tokenomics estimator surfaces the credit/token cost of a build at the moment you choose it — Neptune-aligned, so the spend is a decision, not a surprise.

06

One deploy, many SEs

The discovery cost is amortized across the whole practice. Every SE who onboards consumes the shared library — the more SEs join, the cheaper each one's path to scale.

TaskDefault modelWhy
Transcript → structured outcomesPhi-4-mini~15–20× cheaper; high quality on conversation
Complex reasoning / architecturePhi-4 / GPT-4o-miniescalate only when the small model strains
Hardest, highest-stakes buildsFrontier (GPT-4o)fallback for the rare case that needs it
Indexing, contracts, dedupe, templatesNone (deterministic)code does it — no tokens at all

The ladder auto-selects: cheapest capable path first, escalate on demand, fall back to rules if the model is unavailable — so work never fails and rarely overspends.

The tokenomics thesis: the 2B-token number is a one-time discovery investment, not a per-SE operating cost. MCAPS OS turns that spend into a durable, distributable library — so the practice scales the output (every SE operating at full scale) while the spend per SE trends toward the cost of loading an artifact. Pay it once, distribute the result.
